# ADR-001: Go를 백엔드 언어로 선택

## 상태
승인됨

## 컨텍스트

하이브리드 클라우드 모니터링 및 배포 시스템의 백엔드를 구축하기 위해 프로그래밍 언어를 선택해야 합니다.

### 요구사항
- Docker API 및 Kubernetes API와의 효율적인 통합
- 실시간 메트릭 수집을 위한 높은 동시성 처리 능력
- 여러 클러스터로부터 동시에 데이터를 수집
- WebSocket을 통한 실시간 스트리밍
- 낮은 메모리 사용량 및 빠른 응답 속도
- 배포 및 운영의 용이성

## 결정

백엔드 언어로 **Go (Golang)**를 선택합니다.

### 선택 이유

1. **네이티브 동시성 지원**
   - Goroutines를 통한 경량 스레드
   - 여러 클러스터 동시 모니터링에 최적
   - Channel을 통한 안전한 데이터 공유

2. **Docker/Kubernetes 생태계**
   - Docker와 Kubernetes가 Go로 작성됨
   - `docker/docker`와 `k8s.io/client-go` 공식 SDK 제공
   - 풍부한 예제와 커뮤니티 지원

3. **성능**
   - 컴파일 언어로 빠른 실행 속도
   - 낮은 메모리 사용량
   - GC 최적화로 지연 시간 최소화

4. **배포 용이성**
   - 단일 바이너리로 컴파일
   - 의존성 관리 간편
   - 작은 Docker 이미지 크기 (scratch 이미지 사용 가능)

5. **타입 안정성**
   - 정적 타입 언어
   - 컴파일 타임 에러 검출
   - IDE 지원 우수

## 결과

### 긍정적 영향
- 높은 동시성 처리로 여러 클러스터 효율적 모니터링
- Docker/K8s API 통합이 자연스럽고 안정적
- 빠른 응답 속도로 사용자 경험 향상
- 단일 바이너리 배포로 운영 간소화

### 부정적 영향
- Python이나 Node.js 대비 러닝 커브 존재
- 웹 프레임워크 생태계가 상대적으로 작음
- 에러 핸들링이 다소 장황할 수 있음

### 트레이드오프
- 개발 속도: 약간 느릴 수 있음
- 성능 및 운영 효율성: 크게 향상
- 전체적으로 시스템 요구사항에 더 부합

## 대안

### Python (FastAPI)
**장점:**
- 빠른 개발 속도
- AI/ML 라이브러리 풍부
- 문법이 간결하고 읽기 쉬움

**단점:**
- 동시성 처리가 복잡 (asyncio)
- 성능이 상대적으로 낮음
- 배포 시 의존성 관리 복잡

**선택하지 않은 이유:** 실시간 다중 클러스터 모니터링에서 성능과 동시성이 중요한 요구사항이므로 Go가 더 적합

### Node.js (Express/NestJS)
**장점:**
- 비동기 I/O 네이티브 지원
- JavaScript로 풀스택 가능
- 빠른 개발 속도

**단점:**
- 싱글 스레드 제약
- 타입 안정성 (TypeScript 필요)
- Docker/K8s SDK가 Go만큼 성숙하지 않음

**선택하지 않은 이유:** CPU 집약적인 작업과 다중 코어 활용에서 Go가 우수

### Java (Spring Boot)
**장점:**
- 엔터프라이즈 환경에서 검증됨
- 강력한 타입 시스템
- 풍부한 라이브러리

**단점:**
- 메모리 사용량 높음
- 시작 시간 느림
- 복잡한 설정과 보일러플레이트 코드

**선택하지 않은 이유:** 경량 시스템 구축이 목표이므로 Go가 더 적합
